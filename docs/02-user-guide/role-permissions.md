# 角色与权限说明

当前系统是课程设计和本地演示项目，没有实现登录、用户管理或细粒度权限控制。只要能打开前端页面并访问后端地址，就可以使用同一套功能。

## 使用角色

| 角色 | 使用方式 | 说明 |
|---|---|---|
| 演示人员 | 打开前端 Demo，按 F1-F9 流程演示 | 不需要数据库，适合课堂展示 |
| 开发/维护人员 | 启动前后端，查看 Swagger 和日志 | 需要 Node.js、Docker、Python 环境 |
| 数据构建人员 | 运行 `data_scripts/` 数据链路 | 需要原始 GPS、OSM PBF 和 PostGIS |
| 验收老师 | 查看 PPT、报告、视频和源码 | 可通过 README 了解运行方式 |

## 功能权限

| 功能 | 是否需要登录 | 是否需要后端 | 说明 |
|---|---:|---:|---|
| Demo F1-F9 | 否 | 否 | 使用前端内置演示数据 |
| 真实 F1-F8 | 否 | 是 | 依赖 FastAPI 和 PostGIS 数据 |
| F9 推荐 | 否 | 间接需要 | 基于 F8 返回结果排序 |
| AI 助手本地 RAG | 否 | 是 | 后端读取本地 Markdown 文档 |
| AI 助手 LLM | 否 | 是 | 需要配置 OpenAI-compatible 环境变量 |
| 数据脚本 | 否 | 需要数据库 | 在后端容器或本地 Python 环境中执行 |

## 安全注意事项

- 不要提交 `.env`，只提交 `.env.example`。
- 不要把真实 API Key 写入 README、PPT 或源码注释。
- 当前系统默认面向本地课程演示，不建议直接暴露到公网。
- 如果后续部署到公网，需要增加登录、鉴权、限流和 CORS 白名单。
