# 术语表

本文解释 Urban Taxi Vis 文档和答辩中常见术语。当前术语口径以真实代码和当前 F1-F9 实现为准。

| 术语 | 说明 |
|---|---|
| GPS 轨迹 | 由车辆在不同时间上报的经纬度点组成的移动序列。 |
| 轨迹点 | 单条 GPS 记录，通常包含车辆编号、时间、经度、纬度。 |
| Trip | 一段连续行程。本项目通常根据车辆、时间间隔、数据质量和离线处理结果把轨迹组织为多个 trip。 |
| 原始轨迹 | 直接由 GPS 点按时间连接得到的轨迹折线，可能包含漂移、断点和异常跳变。 |
| Segment | F1 查询中由连续有效点构成的轨迹片段；时间断裂、跳变距离或异常速度会触发新 segment。 |
| 路网匹配轨迹 | 将 GPS 轨迹匹配到道路网络后得到的道路路径，当前来自离线地图匹配结果。 |
| OSM | OpenStreetMap，开放地图数据来源，本项目用于抽取北京区域道路网络。 |
| PostGIS | PostgreSQL 的地理空间扩展，支持空间类型、空间索引和空间查询。 |
| Redis | 内存键值缓存服务，本项目可用于缓存数据集摘要等重复读取结果。 |
| H3 | 一种六边形空间索引系统。当前主要用于 F6 核心区外部流入/流出区域聚合展示，不是 F4 当前后端主接口。 |
| OD | Origin-Destination，起点到终点的流动关系。F5 统计 A/B 区域之间的 OD 转移。 |
| BBox | Bounding Box，矩形经纬度边界，通常由 `min_lon`、`min_lat`、`max_lon`、`max_lat` 表示。 |
| 热力图 | 用颜色强弱表达空间点密度的可视化方式。 |
| 分级着色 | 按数值区间给网格或区域赋予不同颜色。 |
| 经纬度/投影桶网格 | F4 当前后端主路径使用的网格聚合方式，根据 bbox 和网格大小把轨迹点归入网格桶。 |
| 多矩形并集去重 | F3 统计口径：多个矩形同时查询时，同一车辆命中多个矩形也只在总数中计算一次。 |
| 缓冲距离 | 区域命中时向外扩展的容忍距离，用于减少 GPS 漂移或边界误差带来的误判。 |
| 最大转移时间 | F5 中判断 A/B 转移是否合理的时间阈值，过长转移会被过滤。 |
| strict_od | 严格 OD 模式，通常强调起点和终点分别落在指定区域关系中。F6/F8 均有相关口径。 |
| through_flow | F6 的经过流量模式，强调行程经过核心区后的流入或流出关系。 |
| pass_through | F8 的候选模式，强调路线穿行 A/B 区域关系，不局限于严格起终点。 |
| 高频路段 | 在一定时间范围内被大量匹配轨迹经过的道路片段或道路组。 |
| 高频路线 | 起终点区域之间被多条轨迹共同使用的候选路线。 |
| Road token | F8 中用于表示道路边或道路组的序列单元，便于计算路线相似度。 |
| Jaccard 相似度 | 衡量两个 token 集合相似程度的指标，常用于 F8 路线聚类。 |
| 连通分量 | 相似路线图中互相连通的一组候选路线，可作为同一类路线簇。 |
| Medoid 代表路线 | 路线簇中最能代表该簇的真实候选路线，常用于地图展示。 |
| Corridor | F8 返回的候选走廊或路线簇结果，通常包含几何、trip 数和耗时统计。 |
| routes | F8 响应中的兼容字段，前端 F9 可从 `corridors` 或 `routes` 读取候选路线。 |
| p50 耗时 | 耗时中位数，表示一半样本不超过该耗时。F9 `fastest` 策略优先比较该指标。 |
| p90 耗时 | 90 分位耗时，表示 90% 样本不超过该耗时，常用于衡量稳定性。F9 `stable` 策略优先比较该指标。 |
| avg 耗时 | 平均耗时，F8/F9 可用于展示路线整体通行效率。 |
| trip_count | 某条候选路线或道路组包含的行程样本数。样本数越高，统计解释通常越稳健。 |
| F9 策略推荐 | 当前 F9 的真实逻辑：不请求独立后端，而是在 F8 候选路线中按策略排序并高亮推荐路线。 |
| fastest | F9 策略之一：p50 耗时最低优先，平局时 trip_count 更高者优先。 |
| stable | F9 策略之一：p90 耗时最低优先，平局时比较 p50。 |
| frequent_fast | F9 策略之一：综合相对频次和耗时惩罚，兼顾常用程度与速度。 |
| 只读 Demo | 工作台默认演示状态，读取固定导出样例，不要求后端和数据库在线。 |
| mock adapter | `VITE_DEMO_MODE=true` 时前端 axios 使用的模拟接口层，用于完全不访问后端的演示。 |
| 完整后端模式 | 退出只读 Demo 或关闭 mock 后，前端访问 FastAPI、PostGIS 和真实分析接口。 |
| Docker Compose | 多容器编排工具，用于统一启动后端、数据库和缓存服务。 |
| Swagger | FastAPI 自动生成的接口文档页面，默认访问地址为 `http://localhost:8000/docs`。 |
| AI 项目助手 | 基于本地 Markdown 检索和可选 LLM 的问答入口，用于解释项目文档、功能和排障思路。 |
